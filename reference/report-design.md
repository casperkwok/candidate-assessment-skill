# 报告设计参考(视觉语言)

`render_report.py` 的内联 CSS 就是这套 token 的实现;这里留一份说明,便于日后调整。
风格取自一份内部 PRD 的视觉语言:白底极简、靛蓝强调、语义三色、发丝线表格。
**报告本身品牌中性**——不打任何产品品牌/logo,仅借用视觉风格。

## 色板
| 用途 | 值 |
| --- | --- |
| 品牌蓝(章节号/强调/进度) | `#2B54E0`,深 `#1E40AF` |
| 墨色正文 / 深 navy | `#111827` / `#1E293B` |
| 次要文字 / 更浅 | `#6B7280` / `#9CA3AF` |
| 发丝线 / 边框 | `#E9EBF0` / `#E5E7EB` |

| 警示琥珀 / 琥珀底 | `#C77700` / `#FDF3E5` |
| 危险红 / 红底 | `#DC2626` / `#FCEBEA` |
| 信息蓝底 | `#EEF2FF` |

## 组件约定
- **页眉**:中性小标签「内部评估 · 保密」(蓝底圆角),不放任何产品 logo/字标。
- **章节头**:蓝色小号粗体序号(`01`/`02`…)+ 粗体标题。
- **综合分 hero**:conic-gradient 彩环(颜色随分数),环内大数字;右侧评级药丸 + 一句话总结(左竖线引用)。
- **药丸/徽章**:圆角 999px。评级/状态/风险按语义上色(绿/蓝/琥珀/红/灰)。
- **表格**:无竖线,仅发丝横线;表头小号灰色大写间距;单元格留白充足。
- **维度分**:大号分数 + 下方彩色进度条(颜色随分数:≥80 绿、≥65 蓝、≥50 琥珀、否则红)。
- **亮点**:蓝底圆角块列表。
- **面试问题**:浅底卡片 + 蓝色 topic 标签 + 序号。

## 分数配色阈值(与渲染器一致)
`≥80` 绿 · `65–79` 蓝 · `50–64` 琥珀 · `<50` 红。
