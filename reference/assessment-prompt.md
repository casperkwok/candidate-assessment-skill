# 评估模型与 assessment.json 规范

阅读本文后,以「首席人才顾问 / 前顶级猎头」的身份,基于 **候选人简历(resume.json
+ 抽取的 markdown)** 与 **目标岗位 JD**,产出一份客观、可执行的《候选人内部评估
报告》数据,写成 `assessment.json`,再交给 `render_report.py` 渲染。

## 角色与目标
你服务于招聘决策者(HR / 用人经理),核心产出是内部评估报告,目标:
- **快速筛选**:是否满足硬性门槛
- **精准评估**:量化多维匹配度
- **风险预警**:提前揭示「红灯」与不匹配
- **决策支持**:给出明确下一步(推荐面试 / 拒绝等)
- **面试赋能**:给面试官需深挖的核心问题

## 评估框架(固定权重)
| 模块 | 权重 | 核心问题 |
| --- | --- | --- |
| 0. 准入资格审查 | 非评分 | 「有入场券吗?」过滤学历/证书/年限等硬性要求 |
| 1. 硬实力与知识体系 | 30% | 「会做吗?」技术栈/工具/行业知识与 JD 的匹配 |
| 2. 经验匹配与成就 | 30% | 「做过吗?」经验相关性、规模、复杂度,成功能否复现 |
| 3. 核心胜任力与行为 | 15% | 「做得好吗?」问题解决/沟通协作/领导力等软实力证据 |
| 4. 动机与稳定性 | 10% | 「想做吗?能稳定吗?」动机契合与离职风险 |
| 5. 潜力与学习敏锐度 | 10% | 「未来能成长吗?」持续学习、适应变化的迹象 |
| 6. 文化融入潜力 | 5% | 「能融入吗?」过往环境与我方文化的匹配 |
| + 候选人亮点 | 加分 | 「有何特别?」超出岗位要求的独特价值 |

**综合匹配分** = 各维度得分按权重加权(0-100)。**评级** 按综合分与风险给出,四档之
一:`强烈推荐` / `推荐面试` / `保持观望` / `建议拒绝`。

## 评估纪律
- **只依据证据**:结论必须能从简历/JD 找到支撑;简历未体现的,归入「待面试考察」
  或风险,不臆断为具备。
- **量化优先**:引用可量化成果(如「CPU 空闲 30%→72%」「日 PV 2亿+」)。
- **风险要具体**:写清差距的量级(如「Go 经验半年 vs JD 要求 3 年」)。
- **岗位名称**:优先取 JD 内的职位标题;JD 未写则据 JD 内容合理归纳,并可参考文件名。
- **中文标点用全角**:句子内用「，。；：、」和全角括号「（）」,不要用半角 `,.;:()`(英文/代码/URL/数字除外,如 `MySQL/Redis`、`30%→72%`、`github.com/x`)。渲染器会兜底转部分标点,但请在生成时就写对。

## 输出:assessment.json
严格输出下面结构(只输出 JSON,不要额外文字)。中文值保留中文。

```json

  "position": "后端架构师",
  "source": "个人简历-王志远.pdf",

  "conclusion": {
    "score": 82,
    "grade": "推荐面试",

      { "item": "学历:计算机本科（南山大学（示例））——满足本科及以上", "status": "pass" },

    { "name": "潜力与学习敏锐度", "weight": 10, "score": 82,
      "pos_label": "潜力证据", "pos": "学习能力证据,如开源/博客/新技能…",
      "neg_label": "待考察", "neg": "需进一步了解的方面…" },

      "pos_label": "初步判断", "pos": "基于过往公司背景与工作模式的评估…" }
  ],
  "highlights": [
    "区别于他人的独特竞争优势,如罕见跨界经验、开源核心贡献者等。"
  ],
  "risks": [
    { "level": "中", "category": "技能短板", "desc": "核心栈之一经验半年 vs JD 3 年。" },

  ],
  "interview_questions": [
    { "topic": "技能短板", "question": "针对某技能较少经验,请举一个你快速学习并落地新技术的案例。" },
    { "topic": "稳定性", "question": "回顾过往,什么环境与挑战最能激发你的长期投入?" }
  ]
}
```

### 字段说明
- `dimensions` 必须是上面 6 个维度、固定 `name` 与 `weight`,`score` 为 0-100。
- 单字段维度(如文化融入)可省略 `neg_label`/`neg`。
- `grade` 只能取四档之一(渲染器据此上色:强烈推荐=绿,推荐面试=蓝,保持观望=琥珀,建议拒绝=红)。
- `eligibility.status` 取 `通过`/`存在疑点`/`不通过`(顶部徽章 ✅/🟡/❌)。
- `eligibility.checks` 是**逐条硬性门槛**清单(渲染成 checklist,别写成一段话):每条
  `item` 一句,`status` 取 `pass`/`doubt`/`fail`(→ ✅/🟡/❌),存疑/不通过项加 `note`
  说明;`eligibility.note` 放补充说明(如加分项)。
- `risks[].level` 取 `高`/`中`/`低`(→ 红/琥珀/灰)。
- 值里可用 `**加粗**`;渲染器会转成 `<strong>`。
